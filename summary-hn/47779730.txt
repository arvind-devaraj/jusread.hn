Elevated errors on Claude.ai, API, Claude Code  (243 pts, 224 comments)
https://news.ycombinator.com/item?id=47779730

The Hacker News discussion revolves around ongoing elevated error rates on Claude.ai, particularly related to its API and Claude Code, as indicated by user reports of frequent 500 internal server errors. The status site had not yet acknowledged these issues, prompting users to share their experiences and seek alternatives.

A significant portion of the conversation focuses on comparing Claude with Codex, with many users expressing that Codex offers better performance, more generous usage quotas, and overall reliability. Some users noted that they frequently hit quotas on Claude, while Codex allows for more extensive use without hitting limits. There is also debate about the quality of code generated by both models, with opinions varying on which performs better in planning versus execution.

Notable insights include the suggestion that users consider having backup systems in place due to the unreliability of Claude, as well as discussions about the implications of server-side errors and the potential for service degradation during peak usage times. Some users expressed skepticism about the sustainability of Claude's service given its current performance issues, while others highlighted the need for better infrastructure to handle demand.
