Ask HN: Is anyone experimenting with different ways of using LLMs for coding?  (203 pts, 202 comments)
https://news.ycombinator.com/item?id=48771515

The Hacker News discussion revolves around the challenges and frustrations developers face when using large language models (LLMs) like Claude Code and Codex for coding tasks. The original poster expresses a desire for a more fluid coding experience, akin to hand-writing code, rather than the current prompt-response loop that disrupts their flow state. They seek insights into alternative approaches or innovative uses of LLMs in programming.

In the comments, several users share their experiences and experiments with LLMs, highlighting a range of perspectives. Some suggest focusing on clearer specifications to improve model performance, while others express skepticism about the effectiveness of orchestration tools, noting that they often complicate rather than enhance the coding process. There is a consensus that the current interaction model can hinder the flow state, with many users exploring ways to create a more seamless integration of LLMs into their workflows. Notable insights include the idea of developing tools that allow LLMs to operate more autonomously or in a pair programming style, as well as the recognition that human understanding and oversight remain crucial in the coding process despite advancements in AI.
