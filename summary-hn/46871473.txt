Show HN: GitHub Browser Plugin for AI Contribution Blame in Pull Requests  (62 pts, 34 comments)
https://news.ycombinator.com/item?id=46871473

The Hacker News discussion centers around a new GitHub browser plugin designed to track AI contributions in pull requests, allowing developers to see which lines of code were generated by AI versus human edits. The plugin aims to enhance transparency in code contributions, especially as AI-generated code becomes more prevalent.

In the comments, there is a debate about the necessity and effectiveness of the plugin. Some users question why a new tool is needed when standard git blame tools could suffice, while others highlight the importance of preserving metadata about AI contributions, such as the model and prompt used. There are concerns about accountability, with some arguing that ultimate ownership of code should remain with the human submitter, regardless of AI involvement. Others express skepticism about the potential for misuse, fearing that AI-generated code could lead to an influx of low-quality contributions.

Notable insights include the recognition that while AI can produce coherent code, it often requires significant human editing, and the plugin could help teams gauge the quality and risk of AI-generated contributions. Users also discuss the implications of AI in coding practices, emphasizing the need for careful review and the potential for reputational issues in the context of AI-assisted development.
