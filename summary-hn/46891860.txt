A real-world benchmark for AI code review  (52 pts, 28 comments)
https://news.ycombinator.com/item?id=46891860

The Hacker News discussion revolves around a blog post from Qodo, which introduces a benchmark for AI code review tools. The benchmark aims to evaluate the performance of these tools based on their ability to review code effectively, particularly focusing on real, merged pull requests (PRs).

In the comments, there is a mix of skepticism and support. Some users question the validity of the benchmark, suggesting that it may be biased since Qodo, the company behind it, is also a participant. Concerns about pricing and limitations of the tool are prevalent, with several commenters noting that the cost structure may not be feasible for teams with high PR volumes. Others express interest in the need for standardized benchmarks in the evolving field of AI code review.

Notable insights include the recognition that while the benchmark measures whether tools can identify known bugs, it may not address the more pressing needs of teams in the future, such as proving the integrity of code reviews. Additionally, some commenters propose alternative methods for evaluating code quality, emphasizing the importance of distinguishing between good and bad code rather than merely identifying issues.
